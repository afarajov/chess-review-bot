# ChessLens — веб-интерфейс (дизайн-макет)

Статический макет без бэкенда: чистые HTML/CSS/JS, никаких сборщиков и зависимостей.
Весь интерфейс — **на английском**. Открывается любым способом — двойным кликом по
файлу или через любой статический сервер:

```bash
python3 -m http.server 4173 --directory web
# → http://localhost:4173
```

## Страницы

| Файл | Экран |
|---|---|
| `index.html` | Главная: ввод ссылки/ника, модалка с партиями игрока, фичи, Telegram-CTA |
| `review.html` | **Game Review**: доска со стрелками и анимацией ходов, eval-бар, график оценки, список ходов, место под LLM-комментарий |
| `auth.html` | Log in / Sign up (табы) + вход через Telegram |
| `profile.html` | Аккаунт: My games, привязка Chess.com/Lichess, настройки |
| `about.html` | О проекте: как устроен анализ, категории ходов, роадмап, FAQ |

## Структура

```
web/
├── css/style.css      — вся дизайн-система (токены в :root)

```

## Десктопная раскладка Game Review

Две колонки. Левая (доска + управление + график) идёт в потоке страницы. Правая
(`.review-panel`) **прокручивается независимо от левой**: `position: sticky` +
`max-height: calc(100vh - 92px)` + `overflow-y: auto`, поэтому её скролл не двигает
доску, а скролл страницы не двигает панель. Порядок сверху вниз:
**Coach's comment → Moves → Game summary**.

Три секции полностью независимы — ни одна не делит скролл с другой:
- **Coach's comment** (`.coach-card`) — сверху, обновляется на каждом ходу.
- **Moves** (`.move-list`) — своя фиксированная высота на ~10 строк-ходов
  (`max-height: 296px`, строка = 28px) и **собственный** `overflow-y: auto`.
  Список не «раскрывается» на всю партию и не выталкивает итоги вниз.
- **Game summary** (`.summary-card`) — сразу под ходами; чтобы до него добраться,
  не нужно проматывать все ходы (короткий скролл панели, а не списка ходов).

Порядок блоков на десктопе задаётся **порядком в разметке** `review.html`; на мобиле
он другой и переопределяется через `order` в `@media (max-width: 1020px)`, где
`.review-panel` и `.panel-scroll` становятся `display: contents` (растворяются) —
их скролл/высота там не действуют, а карточки встают в общий поток страницы,
и у `.move-list` включается `max-height: 40vh` со своим скроллом. Автоскролл активного
хода (`highlightMoveList()` в `review.js`) прокручивает **только** контейнер
`#move-list`, не трогая ни панель, ни страницу.

## Мобильная раскладка Game Review

На экранах уже 1020px блоки перестраиваются в один столбец в порядке:
шапка → Coach's comment → **горизонтальный eval-бар** → доска → кнопки навигации →
список ходов → график → итоги.
Реализовано в `style.css` через `display: contents` у колонок и `order` у блоков —
менять порядок можно в одном месте (медиазапрос `@media (max-width: 1020px)`).
На десктопе eval-бар вертикальный слева от доски, на мобиле — горизонтальный над ней
(белые растут слева). JS задаёт только процент через CSS-переменную `--fill`,
ориентацию (height/width) выбирает CSS.

## Анимация ходов

При шаге на один ход вперёд/назад фигура плавно скользит со старой клетки на новую
(`animateMove()` в `review.js`, FLIP-приём: transform + transition 160ms).
При прыжках через несколько ходов (клик по списку/графику) позиция меняется мгновенно.

## Значок категории на доске

Ход, сделанный на текущей позиции, помечается кружком категории (★ ! ✓ ?! ? ??)
справа-сверху над фигурой на клетке назначения — как на Chess.com (`.move-badge`).

## Поведение eval-бара

- Стартовая позиция показывает реальную оценку `GAME.start_eval` (+0.5 у белых),
  а не 0.0.
- Заполнение ограничено диапазоном 5–95% (одна клетка доски = 12.5%, проигрывающей
  стороне всегда остаётся ~4/10 клетки) — константа `EVAL_CAP` в `review.js`.
- **Исключение:** если на доске мат (`mate_in != null`), бар заполняется полностью
  (0% или 100%), а подпись показывает `M5`.

## Автоскролл списка ходов

Кнопки навигации/стрелки прокручивают активный ход **только внутри контейнера
`#move-list`**, не трогая прокрутку страницы (`highlightMoveList()` в `review.js`) —
иначе на мобиле страница прыгала к списку ходов при каждом ходе.

## Палитра категорий ходов

| Категория | Значок | Цвет | Токен |
|---|---|---|---|
| Best | ★ | `#2bc47e` | `--cat-best` |
| Excellent | ! | `#8fca44` | `--cat-excellent` |
| Good | ✓ | `#a9b48b` | `--cat-good` |
| Book | ≡ | `#b58863` | `--cat-book` |
| Forced | □ | `#7f97b3` | `--cat-forced` |
| Inaccuracy | ?! | `#e6b23c` | `--cat-inaccuracy` |
| Mistake | ? | `#e0783c` | `--cat-mistake` |
| Blunder | ?? | `#e4453b` | `--cat-blunder` |

Порядок категорий (лучшие → худшие) задаётся в `CAT_LABEL` в `review.js` — он же
определяет порядок строк в «Game summary». Добавить новую категорию = дописать её
в `CAT_LABEL` / `CAT_SYM` / `CAT_COLOR` (review.js) и `.cat-ico.<Name>` (style.css).

Основной акцент — янтарный `--accent: #e8a33d`, тёмный фон `--bg: #0f1218`.
Все цвета определены CSS-переменными в начале `style.css` — менять тему можно в одном месте.

## Как график обрабатывает маты

Оценка (centipawns) переводится в вероятность победы той же формулой, что и в
`analyse_game` (`winrate()` в `review.js`), поэтому кривая плавная, а маты (±10000)
естественно прижимаются к краю. Плюс поверх графика:
- диапазоны ходов с `mate_in != null` закрашиваются штриховкой (зелёная — матуют белые,
  красная — чёрные), широкие зоны подписываются «Mate in N»;
- в тултипе и на eval-баре мат показывается как `M5`, а не `+99.8`.

## Что нужно от бэкенда (точки интеграции)

1. **`js/data.js` → API.** UI читает глобальный объект `GAME` — та же структура, что
   отдаёт `analyse_game()`, плюс поля, которые сейчас добавлены при генерации мока:
   - `san` — ход в человеческой нотации («Nf3», а не «g1f3»);
   - `best` / `best_san` — лучший ход движка в позиции **до** сыгранного хода
     (нужен для зелёной стрелки; в `analyse_game` он уже есть как `info["pv"][0]`,
     просто не пишется в отчёт);
   - `accuracy` и `counts` — точность и число ходов каждой категории по игрокам
     (можно считать на бэке или на клиенте);
   - `start_eval` — оценка стартовой позиции в centipawns (в `analyse_game` это
     `prev_eval` до первого хода, просто не пишется в отчёт);
   - шапка партии: `white/black {name, elo}`, `result`, `date`, `time_control`,
     `opening`, `link`, `source`.
2. **`handleAnalyzeSubmit()` в `common.js`** — сюда подставить вызовы API:
   ссылка → создать анализ и редирект на `review.html?game=…`;
   ник → запросить список партий игрока и отрисовать `renderGameRow()` в модалку.
3. **LLM-комментарий** — контейнер `#coach-comment` на `review.html`.
   Сейчас там скелетон; при смене хода вызывается `renderCoach()` в `review.js` —
   туда же подставлять текст комментария для текущего `ply`.
4. **`js/mock.js`** — заменить `MOCK_RECENT_GAMES` / `MOCK_MY_GAMES` на данные API.

## Управление на странице разбора

`←` / `→` — ходы назад/вперёд, `Home` / `End` — начало/конец партии,
`F` — перевернуть доску. Клик по графику или по ходу в списке — переход к позиции.
Кнопка «⚡ Next mistake» прыгает к ближайшей неточности/ошибке/зевку.
